# Weathr — приложение погоды

**Weathr** —  это современное приложение погоды, построенное на React, TypeScript и Vite. Цель этого pet-проекта — потренироваться в архитектуре (feature-sliced), работе с глобальным состоянием и продуманном UI на основе реального сценария: отображение прогноза погоды для выбранного города. Приложение показывает текущие условия, почасовой и дневной прогноз, позволяет настраивать единицы измерения и использует кастомную тёмную тему. Репозиторий создан для учебных целей и не аффилирован ни с одним реальным погодным сервисом.

Дизайн основан на бесплатном шаблоне **Modern Product Launch** из сообщества Figma. Цель — воспроизвести внешний вид, структуру шаблона и прокачать навыки создания web приложений с использованием новых для меня технологий. Макет: [Modern Product Launch on Figma](https://www.figma.com/community/file/1487309170684591074/modern-product-launch)

Ссылка на [Github pages](https://sunsetteq.github.io/weathr-vite/)

## Обзор
Приложение состоит из одной основной страницы, которая собирается из виджетов в духе FSD архитектуры. Она включает блок текущей погоды, краткий дневной прогноз, почасовой прогноз и панель настроек единиц измерения. 
Глобальное состояние и настройки (единицы температуры, скорости ветра, осадков и т.п.) хранятся в **Redux Toolkit**. Бизнес-логика вынесена в слайсы, а UI-компоненты переиспользуются между блоками. Стайлинг реализован через **Tailwind CSS v4** и **daisyUI** с кастомной тёмной темой на базе OKLCH. 
Разрешение модулей использует кастомные алиасы, настроенные в `vite.config.ts`, чтобы избегать «../../..» в импортах, например `@app` для корневого слоя приложения и `@ui` для переиспользуемых UI-примитивов. Основные данные берутся из публичного погодного API и сервиса геокодирования, обёртки для которых лежат в слое `shared/api`.

## Функции
- **Поиск города и загрузка прогноза** — пользователь может выбрать местоположение и получить для него текущую погоду, короткий прогноз на несколько дней и почасовые значения.
- **Настройки единиц измерения** — переключение:
  - температуры (например, °C/°F),
  - скорости ветра,
  - единиц осадков.  
  - Все настройки хранятся в Redux и применяются ко всем виджетам сразу.
- **Адаптивный интерфейс и тёмная тема** — верстка подстраивается под мобильные, планшеты и десктоп; используется кастомная тёмная тема на базе Tailwind CSS + daisyUI.
- **Типобезопасная архитектура** — весь код написан на TypeScript, используются алиасы и разделение по слоям (app/pages/widgets/features/entities/shared) для удобного рефакторинга.

## Стек технологий

| Инструмент | Назначение |
|---|---|
| **React 19** | Библиотека компонентов для построения интерфейса |
| **TypeScript** | Статическая типизация для более безопасной разработки |
| **Vite** | Быстрый сборщик и dev-сервер |

| **moment** | Работа с датами и временем для отображения прогноза |

| **vite-plugin-svgr** | Импорт SVG как React-компонентов |

## Начало работы

```bash
git clone https://github.com/SunsetTeq/weathr-vite.git
cd weathr-vite

# установка зависимостей
yarn install
# или
npm install

# запуск dev-сервера
yarn dev
# или
npm run dev
```

## Структура проекта
Проект организован по принципам Feature-Sliced Design.  Основные каталоги:
| Каталог | Назначение |
|---|---|
| **src/app** | Настройка глобальных провайдеров и корневой компонент |
| **src/pages** | Верхнеуровневые страницы |
| **src/widgets** | Крупные UI‑блоки (навигация, подвал) |
| **src/shared** | Общие ресурсы: ассеты, конфигурации, хуки, утилиты и UI‑примитивы |
| **src/shared/config/constants** | Конфигурации с данными для страницы |
| **src/shared/assets** | Изображения и иконки, в том числе скриншоты для README |
